Final Step: Start Working on GEF
 


To Compile GEF using 'make' on UNIX or Windows:

  • You will need a make utility that is compatable with GNU make. You can get a version for the PC at Cygnus Solutions.
  • Unzip gefsrc060.zip to get the GEF source code files.
  • Edit the file Makefile.config to set the path to your java compiler and make utility.
  • Set your CLASSPATH to include the following in the given order:
    • classes.zip from your JDK distribution
    • the directory above the uci directory
    • swing-1.0.3/swing.jar
    • xml4j_1_1_14.jar
  • cd into the "uci" directory
  • Type "make"

To Compile GEF on Windows using an IDE (e.g. VisualCafe):

  • Unzip gefsrc060.zip to get the GEF source code files.
  • Make a new empty project and set the CLASSPATH as above. In VisualCafe this is under Project | Options.
  • Add all the GEF source files to your project. One way to do this is to go directory by directory and add all files in each. A faster way to do it is to use the Windows Find utility to find all *.java files, then select them all (Control-A), then drag them into the VisualCafe Project window.

To Run a GEF demo:

  • Make sure that everything is compiled.
  • If you downloaded swing103.zip then you will have to unzip it to get swing-1.0.3/swing.jar
  • Start a new command line window.
  • Set your CLASSPATH to include the following in the given order:
    • classes.zip from your JDK distribution
    • The directory above the uci directory
    • swing-1.0.3/swing.jar
    • xml4j_1_1_14.jar
  • Type the command:
        java uci.gef.demo.BasicApplication
    or on some Windows Java implementations type:
        java -classpath %CLASSPATH% uci.gef.demo.BasicApplication

Keep in Mind:

GEF is a research prototype. It will evolve into a full featured graph editing tool in the future, but it is not comparable to commercial libraries yet. If you believe in the importance of an open-source components, please help in the development of GEF.
